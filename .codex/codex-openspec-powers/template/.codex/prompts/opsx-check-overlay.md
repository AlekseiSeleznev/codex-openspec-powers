Inspect the Codex-native overlay and propose safe repair when generated files or
upgrades removed template-owned behavior.

Steps:

1. Read `.codex/codex-openspec-powers/manifest.yaml`.
2. Read `repairSources.root` from the manifest. For each template-owned target
   except the source bundle itself, derive the bundled source path as
   `<repairSources.root>/<target path>`.
3. Classify files using manifest ownership before proposing any repair.
4. Check required prompts before marker validation.
5. Check required markers, snippets, forbidden paths, optional skill consent
   policy, Superpowers policy, Claude review policy, and documentation
   freshness.
6. For missing or overwritten template-owned files, compare the live file with
   its bundled repair source when available. A required prompt that was
   regenerated by `openspec init --tools codex` is repairable from that source.
7. Report healthy state without writing files.
8. When repair is needed, show affected files, source paths, and exact intended
   changes, then
   ask for confirmation before editing.

Repair must modify only manifest-listed template-owned additions or explicitly
approved merge-review paths. If a live file is fully template-owned and its
bundled source exists, repair by replacing it with that source after
confirmation. For template-owned directories or wildcards, restore missing or
changed files from the bundled source and do not delete user-added files. If
the bundled template source is unavailable, report the missing source and ask
the user to re-apply the template.

**== РЕКОМЕНДАЦИЯ ==**

Repair the listed template-owned overlay additions from
`.codex/codex-openspec-powers/template/`.

Use this only after showing the planned repair scope and confirming it will not
overwrite project-owned content silently.
